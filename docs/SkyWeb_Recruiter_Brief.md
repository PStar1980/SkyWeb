# SkyWeb Recruiter Brief

This document contains short, reusable descriptions of SkyWeb Analytics for recruiters, LinkedIn, job applications, portfolio summaries, and quick interview screens.

## 20-Second Version

SkyWeb Analytics is a full-stack macroeconomic analytics platform I built with React, ASP.NET Core Web API, C#, PostgreSQL, Dapper, Apache ECharts, D3, and Node.js control-plane services. It includes dashboards, saved views, alert rules, a Signal Center, authenticated member preferences, and chart-level alert overlays.

## 45-Second Version

I built SkyWeb Analytics as a portfolio-grade full-stack project that combines macroeconomic data analysis, dashboards, alerts, and modern API architecture. The client is React/Vite with ECharts/D3 charts. The user-facing analytics API is ASP.NET Core/C# with Dapper/Npgsql over PostgreSQL. SkyServer remains a Node.js control plane for ingestion, automation, repository tooling, and alert evaluation. The project demonstrates C#/.NET, React, SQL, API design, data visualization, auth/session handling, and incremental migration discipline.

## LinkedIn Project Post Draft

Built SkyWeb Analytics into a portfolio-ready full-stack macroeconomic analytics platform.

The current stack combines React/Vite, ASP.NET Core Web API, C#, PostgreSQL, Dapper/Npgsql, Apache ECharts, D3, and a Node.js SkyServer control plane. The app supports public macro exploration, authenticated dashboards, saved macro views, alert rules, signal notifications, and chart-level alert overlays.

The most valuable part was the migration path: I moved user-facing API route families from a Node-backed prototype lane into native C# services incrementally, validated each workflow, promoted the .NET lane as the default, and removed the retired client after cutover.

This project now gives me hands-on proof across full-stack development, data visualization, SQL-backed analytics, authentication, dashboard UX, alerting, and architecture tradeoff decisions.

## Safe Boundaries / Do Not Overclaim

Avoid claiming these unless they are implemented later:

- Production cloud deployment.
- Email/browser push delivery.
- Temporal orchestration in production.
- Airflow pipelines.
- Power BI integration.
- GraphQL implementation.
- Public SaaS usage.

Good wording:

> Built as a local-first portfolio platform with production-style architecture and validation practices.

Avoid:

> Deployed enterprise SaaS platform.
